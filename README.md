# CTMC-CKD

This repository contains the code, processed data, and results for a six-state continuous-time Markov chain (CTMC) model of Chronic Kidney Disease (CKD) progression.

The core manuscript in `BMB/BMB.tex` develops a unified CTMC framework for CKD stages 1–5 plus death, with both non-homogeneous and time-homogeneous formulations. Transition rates are modeled via a cause-specific hazard / proportional hazards structure, allowing covariates to enter transition-specific generator matrix entries.

## Project overview

- Disease model: 6-state CTMC with states 1–5 corresponding to CKD stages defined by eGFR thresholds, and state 6 representing death.
- Key modeling assumptions:
  - CKD progression is irreversible and stage-skipping is excluded.
  - Each stage may transition to the next stage or directly to death.
  - Death is an absorbing state.
- Methodology:
  - General non-homogeneous CTMC theory for transition probabilities, sojourn times, and survival quantities.
  - Time-homogeneous baseline model with explicit formulas and likelihood-based parameter estimation.
  - Covariate-dependent transition intensities via a proportional hazards parameterization.
- Data:
  - Synthetic longitudinal electronic health record data generated by Synthea.
  - Model fitting is performed on 335,283 observations from 21,100 synthetic patients.

## Repository structure

- `data_processed/`
  - Processed cohort and panel CSV data used for modeling.
  - Includes `ckd_panel.csv` and `cohort_summary.csv`.
- `results/`
  - Model output files, fitted parameters, evaluation metrics, plots, and summaries.
  - Includes cross-validation results, parameter tables, prevalence outputs, and figures.
- `scripts/`
  - Analysis and modeling scripts.
  - Includes the main model script and cross-validation / sensitivity workflows:
    - `scripts/preprocess.py`
    - `scripts/ckd_msm_M1.R`
    - `scripts/ckd_msm_M1_cv.R`
    - `scripts/ckd_msm_M1_sensitivity.R`
    - `scripts/run_ckd_msm_M1_cv.sh`
    - `scripts/run_ckd_msm_M1_cv_small.sh`

## How to use

1. Review the manuscript source in `BMB/BMB.tex` to understand the model and theoretical details.
2. Inspect `scripts/preprocess.py` to see how raw data are prepared for modeling.
3. Run the R scripts in `scripts/` to fit the CTMC model, perform cross-validation, and generate results.
4. Consult `results/` for estimated parameters, validation metrics, and generated diagnostic plots.
